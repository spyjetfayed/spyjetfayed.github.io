# Introduction

Building the kernel is the first step towards becoming a kernel developer, while it can be achieved with a simple make all command which builds the kernel in the same directory as the source (called In-Tree build), we often would want to have a separate directory for our build artifacts (Out-Of-Tree build or OOT) when you have to build kernels all day and often with different configurations. Not polluting to the source directory is always a good idea.

For this reason I keep multiple build directories each with its own build.sh file which bulids the kernel into that directory. Depending on the build I manually uncomment what I need.

# Getting the source

Subsystems and Maintainers have their own source trees which can be found at https://git.kernel.org/.

Cloning Linus’s tree. 

git clone https://git.kernel.org/pub/scm/linux/kernel/git/torvalds/linux.git/ $HOME/lkmp/linux-trees

# Build Config

The SRC_DIR is where the kernel source lives. The BUILD_DIR is where the build artifacts will go.

ARCH is the architecture we are building for here we choose x64. For a different architecture we can change this to ARM, ARM64. This is not mandatory as the kernel build will default to the host machine architecture. I utilize it for cross-compiling which can be found here.

By default the kernel will use gcc to build but you can choose LLVM by passing LLVM=1 to the make command. To build for 32-bit with gcc you need to use the appropriate compiler flag -m32 and have the appropriate linker you can pass these with the CC and LD parameters.

To show build warnings you pass W=n, where n=1,2,3 depending on what level of errors you want to see.

By default the kernel will check if there is a .config file containing the kernel configuration in the BUILD_DIR, but you can choose your own custom config file with the CUSTOM_CONFIG parameter. 

#!/bin/sh

SRC_DIR=$HOME/lkmp/linux-trees/
BUILD_DIR=$HOME/lkmp/builds/mainline
ARCH=x86_64
# LLVM="LLVM=1"
# CC="gcc -m32"
# LD="ld -m elf_i386"
   
WARNINGS="W=1"     # can set 1,2,3
KCFLAGS='KCFLAGS="-Wno-error"'
VERBOSE="V=1"

NPROC=12
# CUSTOM_CONFIG="KCONFIG_CONFIG=.config.real.hardware"

# Final make command to run
MAKE="make -C $SRC_DIR \
    O=$BUILD_DIR \
    ARCH=${ARCH} \
    $LLVM \
    $CUSTOM_CONFIG \
    $VERBOSE $WARNINGS -j${NPROC}"

# Generating Kernel Config

The kernel config needs to be generated beforehand in the file .config in the BUILD_DIR.

Knowing what kernel configs are required for your particular hardware can be challenging right away. To use your current kernel’s config can use make localmodconfig. This copies the config from /boot or /proc and then trims the config to only include the kernel modules that are currently loaded by the kernel. Depending on which version of the kernel the config came from it will update the .config to use the latest kernel configs and prompt the user for the values. You can choose the default values by pressing Return.

To automate choosing the default option in the script pipe the yes command. 

yes "" | ${MAKE} localmodconfig

It often happens that I would forget to attach some device due to which the kernel module will not get loaded and subsequently localmodconfig will not pick up the module and I would have to redo the build.

To solve that you can save your custom lsmod output and tell localmodconfig to use that instead using the LSMOD= parameter. 

yes "" | ${MAKE} LSMOD=./my-custom-config localmodconfig

There are default configs that come with the linux kernel - kvm_guest.config, tinyconfig, allyesconfig, allmodconfig, x64_86_defconfig. These can be useful for debugging purposes, you can find more details about them with the make help command.

To customize further you can use the gui menu with make nconfig, make menuconfig, make xconfig etc. depending on which gui frontend you prefer.

When just copying the config, lets say from syzbot bug report and building the kernel the latest kernel with the config run make oldconfig or make olddefconfig (chooses default without prompting). This will update the .config with the latest config options.

# Build & Install

# Kernel

make all will build the vmlinux, bzImage and all modules. To build selectively use make vmlinux or make bzImage.

You can instal the kernel using the make install, this will install vmlinuz and System.map into the /boot directory. However, when using INSTALL_PATH=/mnt/boot, to set a custom path it will install in /mnt/boot and not in /mnt/boot/boot.

# Modules

make modules builds all the modules. To build an individual module use make M=drivers/net/ethernet/realtek modules. The built modules are placed in the BUILD_DIR by default but you can change it by adding MO= before M= make MO=/tmp/module M=drivers/net/ethernet/realtek modules.

make modules_install by default installs the modules at / but it will create subdirectories like /lib/modules/<kernel_version>/. So if you set a custom path with INSTALL_MOD_PATH=/mnt it will install at /mnt/lib/modules/<kernel_version>/.
