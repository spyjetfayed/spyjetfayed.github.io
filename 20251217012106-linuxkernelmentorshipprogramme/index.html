# Background

I had been working as an Angular Dev before deciding to apply for the Linux Kernel Mentorship Program, my C knowlege was limited to what was needed to tinker and daily-drive linux for the last six-or-so years and the unwillingness to learn the language too in light of modern languages like Zig, Rust, Odin, D, Nim which all sounded very promising. But eventually my cautious curiosity got the better of me and rest is history. So I will try and make this post as useful for someone with a similar background knowledge.

# Books

The kernel docs although very useful, I did not find to be well structured. There is a lot of stray information which you will need to group together in you head. My initial attempts to understand subsystems was futile as I lacked a background in operating system workings.

The two books that I feel you cannot go wrong with (as of 2025) would be:

  • Hands-On System Programming with Linux
  • Linux Kernel Programming

I found these two books to be the most beginner friendly and recent enough. I won’t consider these to be a substitutes for books by Robert Love, Greg-KH or Michael Kerrisk, even when some of these are more than a decade old at this point. The few pages that I had read still provided me with valuable information.

The good thing about books is that it is very easy to reference something which you forgot. Even when something is outdated in the book it provides a good marker, from where you can go and look up the source or LWN (it is amazing).

There is no substitute for the source code however, I would encourage you to do documentation or backporting work along with this. I made a mistake by not looking into the source as often.h This is something I learn’t about C development, you don’t try too hard to rummage through the docs and blogs to search for an explanation, you look directly look at the source and try to understand and you may often make a wrong assumption (and so do the LLMs!) of what it does, which is part of the learning process, nothing to get frustrated about.

Lastly Javier Carrasco Cruz has a complete list of books you will ever need, only book I would add to it would be Rami Rosen’s Linux Kernel Networking if you were interested in netdev.
