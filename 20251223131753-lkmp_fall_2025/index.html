# Background

Deep diving into a linux system was something that I had wanted to do since in 2022, but it was only this year that while searching for guidance on systems programming on r/kernel I found a link to Mentorship Program run by the Linux Foundation! It happens thrice every year and I was just 2 weeks away from the Fall Session start. I was very excited and anxious after the application process which required completing tasks like building the kernel, adding EXTRAVERSION, reproducing a syzbot bug, etc. as I felt I could have done better. I felt especially lucky when I got an invitation to the program, a Discord Invite and the Zoom meet link.

About my technical side I did not have the slighest idea about Virtual Memory, Paging, Scheduling, Interrupt, Locking. Prior to this my C experience was - the first 20 episodes of the legendary Handmade Hero game development course run by Casey Muratori, first 2 chapters of Robert Love’s Linux System Programming back in 2022 and built a cli tool I built using libxml, curl, xpath libraries. Apart from that my daily work involved javascript and python, so naturaly getting into kernel programming right away might have been more than I had asked for, but things put off long enough go to the fabled Never-land and since I am here writing thing something must have worked out!

# Books

I find books useful for the initial learning phase since they provide well-structured information that is easy to reference later. Although for kernel development books become out-dated, I was happy to find Kaiwan Billimoria’s, Hands-On System Programming with Linux & Linux Kernel Programming, which were comparatively recent and most beginner friendly. All the foundational knowledge required were covered more or less along with github repos for exercises.

For more advanced books you should definitely checkout Javier’s blog.

# FAQ

Disclaimer: These were collected over the course of my mentorship from the Office Hours held weekly and the Discord group discussions and best to my knowledge.

Q. Is it necessary to have the same email account as the signup for Linux Foundation when sending patches? A: No it is not necessary, but what is important is you use your legal name for the patches you send, since you are the legal owner and having your legal name helps verify that easily.

Q. What is the criteria for patches to get accepted for graduation? A: Criteria:

  1. Patch can be a feature, documentation, kernel fix or documentation fix. It needs to be something maintained within the kernel and not maintained externally.
  2. Your patch needs to get merged into any of the branches (linux-stable, linux-next, linux-media, etc. refer here to see all the trees), within the mentorship period. Depending upon circumstances some additional time may be granted.
  3. Grammatical and spelling fixes do not count.

Q. Am I a Linux Foundation employee? A: No, this is a mentorship program to help you get started into the kernel.

Q. If I have a contributed to the kernel before will it count? A: Yes it would.

Q. Can I stream/record the Office Hours meeting? A: No, due to privacy concerns of all the paricipants that is not allowed by the Linux Foundation. But you are free to stream your personal development process away from Office Hours.

The good thing about books is that it is very easy to reference something which you forgot. Even when something is outdated in the book it provides a good marker, from where you can go and look up the source or LWN (it is amazing).

There is no substitute for the source code however, I would encourage you to do documentation or backporting work along with this. I made a mistake by not looking into the source as often.h This is something I learn’t about C development, you don’t try too hard to rummage through the docs and blogs to search for an explanation, you look directly look at the source and try to understand and you may often make a wrong assumption (and so do the LLMs!) of what it does, which is part of the learning process, nothing to get frustrated about.

Lastly only book I would add to it would be Rami Rosen’s Linux Kernel Networking if you were interested in netdev.
